# 小程序真机调试说明

真机调试时出现 **「商城初始化失败」**、**ERR_CONNECTION_REFUSED**，是因为手机访问的接口地址是 `127.0.0.1`，在真机上指向的是手机自己，连不到你电脑上的后端。

## 操作步骤

### 1. 查电脑本机 IP（局域网）

- **Windows**：命令行执行 `ipconfig`，看「无线局域网适配器」或「以太网」下的 **IPv4 地址**，例如 `192.168.1.100`
- **Mac**：系统设置 → 网络 → 高级 → TCP/IP，或终端执行 `ifconfig | grep "inet "`

### 2. 小程序里改成用本机 IP

在项目根目录（与 `package.json` 同级）新建或修改 **`.env`** 或 **`.env.local`**：

```bash
# 把 192.168.1.100 换成你上一步查到的 IP
VITE_API_BASE_URL=http://192.168.1.100:8080
```

保存后**必须重新编译**：先停掉当前的 `npm run dev:mp-weixin`，再重新执行一次。小程序端接口地址是在编译时写死的，不重启不会生效。

### 3. 保证后端可被局域网访问

- 后端已配置为 `server.address: 0.0.0.0`，会监听本机所有网卡。
- 手机和电脑要在**同一 WiFi**（同一局域网）。
- 若电脑有防火墙，需放行 **8080 端口**。

### 4. 微信小程序后台（正式发布时）

正式发布前，在微信公众平台 → 开发 → 开发管理 → 开发设置里，把接口域名配到 **request 合法域名** 和 **uploadFile 合法域名**（需 HTTPS）。真机调试阶段可在开发者工具里勾选「不校验合法域名」临时绕过。

---

**小结**：  
1. 把 `.env` 里的 `VITE_API_BASE_URL` 改成 `http://你的电脑IP:8080`  
2. **停掉并重新运行** `npm run dev:mp-weixin`（否则用的还是旧地址）  
3. 手机和电脑同一 WiFi，后端已启动，再真机调试
