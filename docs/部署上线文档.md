# 部署上线文档（mall）

## 0. 部署目标与建议拓扑

建议至少拆分为 3 类交付物：

- **后端服务**：`mall-server` 打包 Jar，监听 `8080`
- **管理后台**：`mall-admin-web/dist` 静态站点，由 Nginx 托管
- **小程序**：`mall-miniprogram` 构建产物上传到微信小程序平台（并配置合法域名）

典型拓扑：

- `https://admin.<domain>` → Nginx（静态）→ 管理后台
- `https://api.<domain>` → Nginx（反代）→ `http://127.0.0.1:8080`（后端）
- `https://api.<domain>/upload/**` → Nginx（或后端）提供上传图片访问

### 1.1 后端

- **JDK**：17（`mall-server/pom.xml` 指定 `java.version=17`）

### 1.2 前端（构建机）

- Node.js（建议 LTS）
- npm（项目已使用 `package-lock.json`）

## 2. 后端发布（mall-server）

### 2.1 构建产物

在构建机执行：

```bash
cd mall-server
mvn -pl safe-product-module-server -am clean package
```

产物通常位于：

- `mall-server/safe-product-module-server/target/*.jar`

> 说明：该模块使用 `spring-boot-maven-plugin`，默认会生成可运行 Jar。

### 2.2 生产环境配置（必须）

生产 profile 为 `prod`，且 `application-prod.yml` 已关闭自动建表灌数（`spring.sql.init.mode=never`），你需要准备好数据库与表结构。

建议通过环境变量注入（示例）：

```bash
SPRING_PROFILES_ACTIVE=prod
MALL_DB_URL=jdbc:mysql://127.0.0.1:3306/mall_phase?useUnicode=true&characterEncoding=UTF-8&serverTimezone=Asia/Shanghai&allowPublicKeyRetrieval=true&useSSL=false
MALL_DB_USERNAME=mall_user
MALL_DB_PASSWORD=****** 
```

同时建议覆盖上传目录为持久化路径（示例）：

```bash
YUAN_UPLOAD_PATH=/data/mall/upload
```

> 若你希望用环境变量覆盖 `yuan.upload.path`，可采用 Spring 的松散绑定：`YUAN_UPLOAD_PATH` → `yuan.upload.path`。

### 2.2.1 订单“锁库存/支付确认”说明（方案 A）

本项目下单与库存采用“**下单锁库存 → 支付成功确认扣减 → 支付失败/取消/超时释放**”的方案，核心要点：

- 下单创建订单状态为 **`待支付`**，并对商品做 `locked_stock` 预占（防超卖）
- 支付成功后才会：`stock -= qty`、`locked_stock -= qty`、`monthly_sales += qty`，订单进入 **`待发货`**
- 支付失败/用户取消/超时关闭：订单进入 **`已取消`**，并释放 `locked_stock`，锁券回退
- 系统每分钟自动关闭超时未支付订单（基于 `mall_system_setting.cancel_minutes`）

相关表字段与迁移脚本：

- `mall_goods.locked_stock`
- `mall_order.coupon_id/pay_deadline/pay_time/cancel_time`
- 生产迁移脚本：`mall-server/safe-product-module-server/src/main/resources/sql/migration-lock-stock-and-order-pay.sql`

### 2.2.2 鉴权与购物车上线注意事项

- 小程序端除公开接口外均需携带请求头：
  - `X-User-Id`
  - `X-Token`
- 登录接口会更新用户 token（简化版），默认有效期 7 天
- 购物车已改为服务端持久化（表：`mall_cart`），上线迁移需执行建表 SQL

### 2.2.3 支付回调日志与幂等

- 表：`mall_pay_log`（`trade_no` 唯一索引）
- 用途：
  - 落库回调原始信息（占位）
  - 回调重放通过 `trade_no` 去重，配合订单状态幂等保证不会重复扣减/重复释放

如需上线迁移 Banner 扩展字段（图片/跳转/启停），执行：

- `mall-server/safe-product-module-server/src/main/resources/sql/migration-banner-crud.sql`

### 2.3 启动方式（Linux，推荐 systemd）

1) 将 Jar 上传到服务器，例如：

- `/opt/mall/mall-server.jar`

2) 创建目录：

- `/data/mall/upload`（与 `yuan.upload.path` 对应）

3) 新建 systemd 服务（示例：`/etc/systemd/system/mall-server.service`）：

```ini
[Unit]
Description=mall-server
After=network.target

[Service]
Type=simple
WorkingDirectory=/opt/mall
Environment=SPRING_PROFILES_ACTIVE=prod
Environment=MALL_DB_URL=jdbc:mysql://127.0.0.1:3306/mall_phase?useUnicode=true&characterEncoding=UTF-8&serverTimezone=Asia/Shanghai&allowPublicKeyRetrieval=true&useSSL=false
Environment=MALL_DB_USERNAME=mall_user
Environment=MALL_DB_PASSWORD=******
Environment=YUAN_UPLOAD_PATH=/data/mall/upload
ExecStart=/usr/bin/java -jar /opt/mall/mall-server.jar
Restart=always
RestartSec=3

[Install]
WantedBy=multi-user.target
```

4) 启动：

```bash
sudo systemctl daemon-reload
sudo systemctl enable mall-server
sudo systemctl restart mall-server
sudo systemctl status mall-server
```

### 2.4 健康检查与验收

- `GET /mall/admin/dashboard`（管理端工作台统计）
- `GET /mall/mini/category/list`（小程序分类）
- 图片访问：`GET /upload/<file>`（需确保上传目录有文件）

订单闭环验收（演示接口）：

- 下单：`POST /mall/mini/order/create`
- 支付预下单（占位）：`POST /mall/mini/pay/prepay`
- 支付回调（占位）：`POST /mall/mini/pay/notify`
- 标记支付成功（测试回调）：`POST /mall/mini/order/pay/success`
- 标记支付失败（测试回调）：`POST /mall/mini/order/pay/fail`
- 用户取消订单：`POST /mall/mini/order/cancel`

## 3. 管理后台发布（mall-admin-web）

### 3.1 构建

```bash
cd mall-admin-web
npm ci
```

设置生产接口地址（建议使用构建时环境变量），例如在构建机注入：

```bash
# Windows PowerShell 示例
$env:VITE_API_BASE_URL="https://api.<domain>"
npm run build
```

产物目录：

- `mall-admin-web/dist/`

### 3.2 Nginx 静态站点示例

```nginx
server {
    listen 80;
    server_name admin.<domain>;

    root /var/www/mall-admin;
    index index.html;

    location / {
        try_files $uri $uri/ /index.html;
    }
}
```

将 `dist/` 内容同步到 `/var/www/mall-admin/` 即可。

## 4. API 域名与反向代理（建议使用 Nginx）

示例（`api.<domain>`）：

```nginx
upstream mall_server {
    server 127.0.0.1:8080;
}

server {
    listen 80;
    server_name api.<domain>;

    client_max_body_size 10m;

    location / {
        proxy_pass http://mall_server;
        proxy_set_header Host $host;
        proxy_set_header X-Real-IP $remote_addr;
        proxy_set_header X-Forwarded-For $proxy_add_x_forwarded_for;
        proxy_set_header X-Forwarded-Proto $scheme;
    }
}
```

> 若上生产，建议开启 HTTPS（证书申请/续期按你们现网规范执行）。

## 5. 小程序发布（mall-miniprogram）

### 5.1 构建

```bash
cd mall-miniprogram
npm ci
npm run build:mp-weixin
```

### 5.2 微信公众平台域名配置（必须 HTTPS）

在微信公众平台 → 开发 → 开发管理 → 开发设置：

- **request 合法域名**：`https://api.<domain>`
- **downloadFile 合法域名**：`https://api.<domain>`
- **uploadFile 合法域名**：`https://api.<domain>`

否则线上将出现：

- 接口请求失败
- 图片无法下载显示（`/upload/**`）

## 6. 数据库上线要点

- 开发环境会自动执行 `sql/schema-mysql.sql`、`sql/data-mysql.sql`
- 生产环境默认不自动初始化（`prod` 配置为 `never`）

上线建议流程：

- 首次上线：由 DBA 执行 `schema-mysql.sql` 建表（必要时结合 `migration-add-category-icon.sql`）
- 后续变更：以迁移脚本/版本化方案执行，避免应用启动时隐式改表

---

## 7. 回滚策略（建议）

- **后端**：保留上一版 `mall-server.jar`，systemd 指向回滚 Jar 后重启即可
- **管理后台**：保留上一版静态目录（或用版本号目录 + 软链切换）
- **上传文件**：`/data/mall/upload` 作为持久化目录，不随发布回滚删除
- **数据库**：变更必须有对应回滚脚本/备份策略（按你们 DBA 规范）

---

## 8. 常见线上问题排查

### 8.1 图片 404

- 检查上传目录是否一致：`yuan.upload.path`
- 检查 `/upload/**` 的映射是否对外可访问（API 域名/反代是否放行）

### 8.2 小程序图片/请求被拦截

- 检查微信公众平台是否配置了 **HTTPS** 合法域名（request / downloadFile / uploadFile）
