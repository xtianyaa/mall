# 工程说明文档（mall）

## 1. 项目概述

本仓库为**微信商城首期**工作区，目标是完成“浏览商品 → 加购 → 下单 → 订单管理”的核心交易闭环，并配套管理后台与后端服务。

- **小程序端**：`mall-miniprogram`（`uni-app + Vue3 + TypeScript`）
- **管理后台**：`mall-admin-web`（`Vue3 + TypeScript + Element Plus + Pinia`）
- **后端服务**：`mall-server-backend/mall-server`（`Spring Boot + MyBatis-Plus + MySQL`，多模块）

---

## 3. 子项目说明

### 3.1 `mall-server-backend/mall-server`（后端）

#### 3.1.1 模块划分

见 `mall-server-backend/README.md` 与 `mall-server-backend/mall-server/README.md`，模块如下：

- `mall-server`：启动模块（`MallServerApplication`）
- `mall-web`：Controller、异常处理、Web 配置（含静态资源映射）
- `mall-service`：DTO/VO、服务接口、统一返回结构
- `mall-core`：核心业务实现
- `mall-persistence`：Entity、Mapper（MyBatis-Plus）

#### 3.1.2 关键配置（端口/环境/上传）

`mall-server-backend/mall-server/src/main/resources/application.yml`：

- **端口**：`server.port=8080`
- **监听地址**：`server.address=0.0.0.0`（便于真机/局域网联调）
- **上传目录**：`yuan.upload.path=./data/upload`
- **上传限制**：单文件/请求体 `10MB`
- **Profile**：默认 `spring.profiles.active=dev`

`application-dev.yml`：

- **MySQL 连接**：通过环境变量拼接（带默认值）
  - `MALL_DB_HOST`（默认 `127.0.0.1`）
  - `MALL_DB_PORT`（默认 `33306`）
  - `MALL_DB_NAME`（默认 `mall_phase`）
  - `MALL_DB_USERNAME`（默认 `root`）
  - `MALL_DB_PASSWORD`（默认 `33taotao!`）
- **初始化 SQL**：启动时自动执行
  - `classpath:sql/schema-mysql.sql`
  - `classpath:sql/data-mysql.sql`

`application-prod.yml`：

- **MySQL 连接**：使用显式环境变量
  - `MALL_DB_URL`
  - `MALL_DB_USERNAME`
  - `MALL_DB_PASSWORD`
- **关闭自动建表/灌数**：`spring.sql.init.mode=never`（生产建议由 DBA/迁移脚本管理）

#### 3.1.3 上传与图片访问（分类图标/商品图片共用）

上传与图片 URL 逻辑详见 `docs/image-url-logic.md`，要点如下：

- 上传接口：`POST /mall/admin/upload/image`
- 服务端存盘：`{yuan.upload.path}/{UUID}.{ext}`
- 返回给前端：`/upload/{UUID}.{ext}`（相对路径）
- 静态映射：`/upload/**` → `file:{yuan.upload.path}/`

> 因此分类图标、商品图片都只需要存 `/upload/xxx`，前端拼成完整 URL 即可。

### 3.2 `mall-admin-web`（管理后台）

#### 3.2.1 本地开发

- 环境变量：参考 `mall-admin-web/.env.example`
  - `VITE_API_BASE_URL=http://127.0.0.1:8080`
- 启动：

```bash
cd mall-admin-web
npm install
npm run dev
```

默认本地端口：`5173`（见 `mall-admin-web/vite.config.ts`，`host: true` 允许局域网访问）。

#### 3.2.2 构建产物

```bash
cd mall-admin-web
npm run build
```

产物目录：`mall-admin-web/dist/`（用于静态部署）。

### 3.3 `mall-miniprogram`（小程序端）

#### 3.3.1 环境变量与真机调试

- 环境变量：`.env` / `.env.example`
  - `VITE_API_BASE_URL=...`
- 真机调试指引：`mall-miniprogram/真机调试说明.md`
  - 真机不能用 `127.0.0.1`，需改成**电脑局域网 IP**（如 `http://192.168.1.100:8080`）
  - 修改后必须重新编译（`uni-app` 会在编译期写死该值）

#### 3.3.2 启动/构建（微信小程序）

```bash
cd mall-miniprogram
npm install
npm run dev:mp-weixin
```

构建：

```bash
cd mall-miniprogram
npm run build:mp-weixin
```

## 5. 常见问题速查

### 5.1 小程序真机提示“商城初始化失败/ERR_CONNECTION_REFUSED”

- 检查 `.env` 的 `VITE_API_BASE_URL` 是否还是 `127.0.0.1`
- 改为电脑局域网 IP 后，**停止并重启** `npm run dev:mp-weixin`
- 确保手机与电脑同一 WiFi，电脑防火墙放行 `8080`

### 5.2 分类图标不显示、商品图正常

优先看 `docs/image-url-logic.md` 的排查章节：

- 分类 `icon` 为空时后端会返回 emoji（不是图片 URL）
- 如需网络图片，必须在后台为分类上传并保存图标（保存 `/upload/xxx`）

## 6. 关键业务实现说明

### 6.1 小程序端鉴权

- 登录接口：`POST /mall/mini/login` 会下发 `token` 与 `tokenExpireTime`
- 后续小程序接口通过请求头携带：
  - `X-User-Id`
  - `X-Token`
- 后端对 `/mall/mini/**` 做统一拦截校验（公开接口如 banner/goods/category/login 会放行）

### 6.2 购物车

- 表：`mall_cart`
- 接口：
  - `GET /mall/mini/cart/list`
  - `POST /mall/mini/cart/add`
  - `POST /mall/mini/cart/update`
  - `POST /mall/mini/cart/remove`

小程序侧逻辑：

- 未登录：本地缓存
- 已登录：与后端同步，登录时会把本地购物车合并到服务端，避免丢车

### 6.3 下单与金额校验

真实项目下单不信任前端传入金额与价格，后端按 `goodsId + quantity` 查库重算：

- `totalAmount`：按数据库商品单价汇总
- `payAmount`：按优惠券规则计算折扣后得出
- 同时校验收货地址必须属于当前用户

推荐下单入口：

- `POST /mall/mini/order/createFromCart`：从购物车勾选项创建订单，成功后会清理购物车已结算条目
