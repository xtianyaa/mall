# 商品下单与售后完整流程说明

## 一、订单状态与流转

### 1.1 订单状态枚举

| 状态       | 说明                         |
|------------|------------------------------|
| 待支付     | 下单未支付，可取消/关单      |
| 支付处理中 | 支付回调处理中（短暂态）     |
| 待发货     | 已支付，等待商家发货         |
| 配送中     | 已填写物流，配送中           |
| 已完成     | 用户签收或管理端确认完成     |
| 已取消     | 用户取消、超时关单或售后退款关闭 |

### 1.2 下单与订单主流程

```
用户下单 → 待支付
    ↓ 支付成功
支付处理中 → 待发货
    ↓ 管理端发货（填写快递）
配送中
    ↓ 管理端「确认完成」或后续签收逻辑
已完成

分支：
- 待支付 → 用户取消 / 超时关单 / 管理端关单 → 已取消
- 待发货/配送中 → 用户申请「仅退款」且审核通过/完成 → 已取消（本次修复）
```

- **创建订单**：小程序下单，订单状态为「待支付」，锁定库存。
- **支付**：支付回调成功后订单变为「待发货」，扣减库存。
- **发货**：管理端在订单列表对「待发货」订单发货，填写快递公司与单号，订单变为「配送中」。
- **完成**：管理端对「配送中」订单执行「确认完成」，订单变为「已完成」。
- **取消**：仅「待支付」可被取消/关单，变为「已取消」。

## 二、售后流程

### 2.1 售后类型

| 类型           | 说明           |
|----------------|----------------|
| REFUND_ONLY    | 仅退款         |
| RETURN_REFUND  | 退货退款       |
| EXCHANGE       | 换货           |

### 2.2 售后状态

| 状态      | 说明           |
|-----------|----------------|
| APPLIED   | 用户已申请     |
| APPROVED  | 管理端已通过   |
| REJECTED  | 管理端已拒绝   |
| COMPLETED | 售后已完成     |
| CLOSED    | 已关闭         |

### 2.3 售后与订单的联动（重要）

- **仅退款（REFUND_ONLY）**
  - 用户可在订单详情对某一订单项发起「仅退款」申请。
  - 管理端在「售后订单」中可对该售后单进行：**通过（APPROVE）**、**拒绝（REJECT）**、**完成（COMPLETE）**。
  - **修复前问题**：仅退款审核通过或完成后，原订单状态仍为「待发货」或「配送中」，不合理。
  - **修复后逻辑**：当售后类型为 `REFUND_ONLY` 且管理端操作为 **APPROVE** 或 **COMPLETE** 时，若原订单当前状态为「待发货」或「配送中」，则将该订单更新为 **已取消**，并写入取消时间。这样订单不再显示为「配送中」，与「仅退款已成立」的语义一致。

- **退货退款（RETURN_REFUND） / 换货（EXCHANGE）**
  - 当前实现仅更新售后单状态，不自动变更订单状态；如需「退货完成并退款后自动关单」等，可在此基础上再扩展。

### 2.4 售后操作入口

- **小程序端**：订单详情 → 某订单项「申请售后」→ 选择售后类型（仅退款/退货退款/换货）→ 提交后可在「售后详情」查看进度。
- **管理端**：售后订单列表 → 筛选/查看售后详情 → 处理售后（通过/拒绝/完成），仅退款通过或完成时会联动将对应订单置为「已取消」。

## 三、数据与代码位置

- 订单表：`mall_order`，状态字段 `status`。
- 售后主单：`mall_after_sale_order`，关联 `order_id`，字段 `type`、`status`。
- 售后明细：`mall_after_sale_item`，关联订单项 `order_item_id`。
- 售后处理与订单联动逻辑：`AfterSaleServiceImpl.handleAfterSale`（仅退款通过/完成时更新订单为已取消）。

---

## 四、小结

- 下单流程：待支付 → 待发货 → 配送中 → 已完成；取消仅支持待支付 → 已取消。
- 仅退款审核通过或完成后，对应订单若为「待发货」或「配送中」，会自动变为「已取消」，避免仍显示「配送中」的不合理情况。
- 退货退款、换货目前不自动改订单状态，可按业务需要再增加联动规则。
