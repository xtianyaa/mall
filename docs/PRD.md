# 微信商城项目产品需求文档 (PRD)

## 1. 文档概述

### 1.1 文档目的
本文档旨在明确微信商城项目的详细需求，包括功能需求、非功能需求、用户角色、系统流程及数据交互逻辑。作为产品开发、测试及验收的依据，确保项目团队对产品目标与实现细节达成一致。

### 1.2 目标读者
- 产品经理
- UI/UE 设计师
- 前端/后端开发工程师
- 测试工程师
- 项目干系人

### 1.3 项目背景
本项目旨在构建一个基于微信小程序的电商交易闭环系统，配套完善的后台管理系统。首期目标是跑通核心交易流程（浏览-加购-下单-支付-发货-收货），为后续营销、分销等高级功能奠定基础。

---

## 2. 产品概述

### 2.1 产品定位
一款轻量级、高性能的微信小程序商城，为用户提供便捷的商品浏览与购买体验，同时为运营人员提供高效的商品与订单管理工具。

### 2.2 用户角色

| 角色 | 描述 | 主要职责 |
| :--- | :--- | :--- |
| **C端用户** | 小程序访客/买家 | 浏览商品、搜索商品、加入购物车、下单支付、查看订单、管理地址、使用优惠券。 |
| **B端管理员** | 平台运营人员 | 管理商品分类、上下架商品、处理订单发货、查看经营数据、配置营销活动（Banner/优惠券）、系统设置。 |

### 2.3 系统架构
项目包含三个核心子系统：
1.  **小程序端 (`mall-miniprogram`)**: 基于 uni-app + Vue3 开发，运行于微信环境。
2.  **管理后台 (`mall-admin-web`)**: 基于 Vue3 + Element Plus 开发，PC 端访问。
3.  **服务端 (`mall-server-backend`)**: 基于 Spring Boot + MyBatis-Plus，提供统一 API 服务与数据支撑。

## 3. 功能需求说明

### 3.1 小程序端 (C端)

#### 3.1.1 首页
- **Banner 轮播**: 展示后台配置的营销活动图，支持点击跳转。
- **搜索入口**: 顶部常驻搜索框，点击进入搜索页。
- **快捷导航**: 热门分类图标入口（如新品、热销等）。
- **今日推荐**: 展示精选商品列表。
- **交互**: 支持下拉刷新获取最新数据。

#### 3.1.2 商品分类
- **分类导航**: 左侧一级分类列表，点击切换。
- **商品展示**: 右侧展示对应分类下的商品列表或二级分类（当前版本主要展示商品列表）。
- **筛选/排序**: 支持按价格、销量等基础排序（预留）。

#### 3.1.3 搜索
- **关键词搜索**: 输入商品名称/描述进行模糊搜索。
- **热门搜索**: 展示系统预设的热搜词。
- **历史搜索**: 本地存储并展示用户的历史搜索记录。

#### 3.1.4 商品详情
- **基本信息**: 轮播图、商品名称、价格（原价/现价）、销量、库存。
- **商品卖点**: 简短的营销文案展示。
- **图文详情**: 富文本展示商品详细介绍与图片。
- **推荐商品**: 底部展示同分类或相关推荐商品。
- **底部操作栏**: 首页入口、客服入口、加入购物车、立即购买。

#### 3.1.5 购物车
- **列表展示**: 展示已加购商品，区分有效与失效商品。
- **操作**:
    - **勾选**: 单选/全选商品。
    - **数量调整**: 增加/减少购买数量，支持输入。
    - **删除**: 左滑或点击删除按钮移除商品。
- **结算栏**: 展示合计金额，点击“去结算”进入订单确认页。
- **逻辑**: 未登录状态支持本地缓存购物车，登录后自动合并至服务端。

#### 3.1.6 结算与下单
- **收货地址**: 选择或新增收货地址。
- **商品清单**: 展示待支付商品及数量。
- **优惠券**: 选择可用优惠券，自动计算抵扣金额。
- **金额明细**: 展示商品总额、运费、优惠金额、实付金额。
- **提交订单**: 生成订单并唤起支付流程。

#### 3.1.7 支付与订单
- **支付流程**: 模拟支付（首期）或微信支付（规划中），支持支付成功/失败/取消的回调处理。
- **订单列表**: 按状态（全部/待付款/待发货/待收货/已完成/已取消）筛选查看。
- **订单详情**: 展示订单状态、物流信息（模拟）、商品明细、金额明细、下单时间等。
- **操作**: 取消订单（待付款）、去支付、确认收货；对订单项可**申请售后**（仅退款/退货退款/换货），并可在售后详情页查看进度。

#### 3.1.8 用户中心 (我的)
- **个人信息**: 展示头像、昵称（登录后）。
- **订单入口**: 快速查看各状态订单。
- **常用功能**: 地址管理、我的优惠券、联系客服。
- **登录/注册**: 微信授权一键登录。

---

### 3.2 管理后台 (B端)

#### 3.2.1 工作台
- **数据概览**: 今日订单数、今日成交额、待发货订单数、优惠券领取数。
- **趋势图表**: 订单走势、热销商品排行（Top 10）。

#### 3.2.2 商品管理
- **分类管理**:
    - 列表展示：层级结构、图标、排序。
    - 操作：新增、编辑、删除、启用/停用。
- **商品列表**:
    - 筛选：按分类、关键词、上架状态筛选。
    - 操作：新增商品、编辑详情、上架/下架、删除、设置库存/价格。
    - 编辑详情：支持上传主图、轮播图、富文本详情。

#### 3.2.3 订单管理
- **订单列表**:
    - 筛选：按订单号、用户、状态、时间范围筛选。
    - 展示：订单基础信息、实付金额、状态、售后笔数。
- **订单详情**: 查看收货人信息、商品清单、费用明细、物流信息。
- **发货处理**: 填写物流公司与单号，更新订单状态为“配送中”。
- **其他操作**: 关闭订单（待支付）、确认完成（配送中→已完成）、查看物流。

#### 3.2.4 售后订单
- **售后列表**: 按售后类型、状态、时间筛选。
- **售后详情**: 查看售后单号、类型、状态、申请金额、关联订单、处理日志。
- **处理售后**: 对申请执行通过、拒绝或完成；仅退款通过/完成时，系统自动将对应订单置为已取消并释放该订单使用的优惠券。

#### 3.2.5 用户管理
- **用户列表**: 展示用户昵称、手机号（如有）、注册时间、最后登录时间。
- **用户详情**: 查看该用户的收货地址列表、历史订单记录、持有优惠券。

#### 3.2.6 营销管理
- **优惠券管理**:
    - 列表：查看所有优惠券模板及可用时间范围（开始时间～结束时间）。
    - 新增/编辑：设置名称、面额、使用门槛、**可用开始时间**（不填则立即生效）、**可用结束时间**（不填则默认 30 天后）、投放状态。
    - 发放记录：通过用户优惠券台账查看领取与使用情况。
- **Banner 管理**:
    - 首页轮播图配置，设置图片、跳转链接（商品/活动页）、排序权重、启用状态。

#### 3.2.7 系统设置
- **基础设置**: 商城名称、客服电话、LOGO 设置。
- **交易设置**: 订单自动取消时间（如30分钟）、包邮门槛设置。

## 4. 非功能需求

### 4.1 性能要求
- **响应时间**: 核心接口（首页、详情、下单）平均响应时间 < 500ms。
- **并发能力**: 支持基础的并发访问，保证促销活动期间系统稳定性。
- **图片加载**: 图片资源需经过压缩与CDN加速（建议），首屏加载无明显卡顿。

### 4.2 安全要求
- **鉴权机制**:
    - 小程序：基于 JWT (JSON Web Token) 的身份认证，Token 有效期管理。
    - 管理后台：账号密码加密存储，RBAC 权限控制（预留）。
- **数据安全**: 敏感数据（如支付信息、用户隐私）传输需加密（HTTPS）。
- **输入校验**: 所有后端接口需进行严格的参数校验，防止 SQL 注入与 XSS 攻击。

### 4.3 数据一致性
- **库存扣减**: 下单时需保证库存扣减的原子性，防止超卖。
- **金额计算**: 订单金额计算逻辑需在后端统一处理，前端仅做展示，严禁信任前端传递的价格参数。

## 5. 业务流程

### 5.1 核心交易流程（浏览 → 收货）

整体链路如下，供产品、开发、测试对齐主路径与异常分支。

```
[用户] 浏览/搜索商品 → 加购或立即购买 → 选择地址与优惠券 → 提交订单
         ↓
[系统] 创建订单（待支付）、锁定库存、若使用优惠券则锁定该张券（locked）
         ↓
[用户] 去支付 → 支付成功/失败/取消
         ↓
[系统] 支付成功：扣减库存、优惠券 locked→used、订单 待支付→待发货
        支付失败/取消/超时：释放库存、优惠券 locked→unused、订单→已取消
         ↓
[管理端] 对「待发货」订单填写快递公司与单号 → 发货
         ↓
[系统] 订单 待发货→配送中
         ↓
[管理端] 对「配送中」订单执行「确认完成」（或后续接入签收逻辑）
         ↓
[系统] 订单 配送中→已完成
```

**要点**：
- 下单时仅**锁定**库存与优惠券，不扣减；支付成功后才扣减库存并将券标记为已使用。
- 订单取消（待支付下用户取消、超时关单、管理端关单）时，**释放锁定的库存与优惠券**，券可再次使用。
- 若订单曾使用优惠券，在**仅退款审核通过/完成**导致订单变为已取消时，该张券也会释放为未使用。

---

### 5.2 订单状态与流转

| 订单状态   | 说明 |
|-----------|------|
| 待支付     | 下单未支付，可取消/关单 |
| 支付处理中 | 支付回调处理中（短暂态） |
| 待发货     | 已支付，等待商家发货 |
| 配送中     | 已填写物流，配送中 |
| 已完成     | 管理端确认完成或签收 |
| 已取消     | 用户取消、超时关单或售后退款关闭 |

**主流程**：下单 → 待支付 → 支付成功 → 待发货 → 管理端发货 → 配送中 → 管理端确认完成 → 已完成。

**取消/关单**：
- **待支付**：用户取消、超时自动关单、管理端关单 → 已取消；释放库存与优惠券（若有）。
- **待发货/配送中**：用户对某订单项申请「仅退款」，管理端在售后订单中**通过**或**完成**该售后单 → 原订单变为**已取消**，并释放该订单使用的优惠券（若有）。

---

### 5.3 售后流程

**售后类型**：仅退款（REFUND_ONLY）、退货退款（RETURN_REFUND）、换货（EXCHANGE）。

**售后状态**：用户已申请（APPLIED）→ 管理端已通过（APPROVED）/已拒绝（REJECTED）/已完成（COMPLETED）/已关闭（CLOSED）。

**流程简述**：
1. **用户侧**：订单详情 → 选择某订单项「申请售后」→ 选择类型（仅退款/退货退款/换货）→ 提交；可在「售后详情」页查看进度与日志。
2. **管理端**：售后订单列表 → 筛选/查看详情 → 处理售后（通过/拒绝/完成）。
3. **与订单联动**：
   - **仅退款**：管理端对该售后单执行「通过」或「完成」时，若原订单为待发货或配送中，系统自动将订单更新为**已取消**，并释放该订单使用的优惠券。
   - **退货退款/换货**：当前仅更新售后单状态，不自动变更订单状态；后续可按业务扩展（如退货完成并退款后关单）。

---

### 5.4 优惠券流程

**可用时间范围**：每张优惠券具备**可用开始时间**与**可用结束时间**（不填开始时间表示立即生效，不填结束时间则默认 30 天后）。用户在下单选择优惠券及系统校验时，均会校验当前时间是否落在该范围内。

**用户券状态**：未使用（unused）→ 下单占用时锁定（locked）→ 支付成功后已使用（used）。  
订单取消时（待支付取消/超时/管理端关单，或仅退款通过/完成导致订单取消）：将对应那张券从 locked 或 used 释放回 **unused**，可再次使用。

**业务规则**：
- 下单时：仅当券为 unused、在可用时间范围内、满足门槛时可用；选券后占位为 locked。
- 支付成功：该订单占用的券 locked → used。
- 订单取消（任意取消路径）：该订单占用的券释放为 unused（更新一张，LIMIT 1）。

---

### 5.5 库存与支付

- **库存**：下单时**锁定**（locked_stock 增加），支付成功时**扣减**（stock 与 locked_stock 同时减少）；取消/超时关单时仅**释放锁定**（locked_stock 减少）。
- **支付**：首期为模拟支付；上线后对接微信支付，支付成功/失败/取消通过回调或前端结果更新订单状态并驱动库存与优惠券的上述逻辑。
- **超时关单**：系统可定时扫描「待支付」且已过支付截止时间的订单，自动关单并释放库存与优惠券。

## 6. 数据模型核心实体

| 实体 | 说明 | 关键字段 |
| :--- | :--- | :--- |
| **User** | 用户 | id, openid, nickname, avatar, mobile, create_time |
| **Category** | 商品分类 | id, name, icon, parent_id, sort, status |
| **Goods** | 商品 | id, category_id, name, price, stock, detail, status, images |
| **Cart** | 购物车 | id, user_id, goods_id, quantity, create_time |
| **Order** | 订单 | id, order_no, user_id, coupon_id（可选）, total_amount, pay_amount, status, address_text, pay_time, cancel_time, express_company, express_no, ship_time |
| **OrderItem** | 订单明细 | id, order_id, goods_id, goods_name, price, quantity |
| **Coupon** | 优惠券模板 | id, name, amount, min_amount, start_time（可用开始，NULL 即生效）, expire_time（可用结束）, receive_count, status |
| **UserCoupon** | 用户优惠券 | id, user_id, coupon_id, status（unused/locked/used）, expire_time |
| **Address** | 收货地址 | id, user_id, name, mobile, province, city, district, detail, is_default |
