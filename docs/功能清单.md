## 三、下单与售后流程说明

### 3.1 订单状态与主流程

| 订单状态   | 说明 |
|-----------|------|
| 待支付     | 下单未支付，可取消/关单 |
| 支付处理中 | 支付回调处理中（短暂态） |
| 待发货     | 已支付，等待商家发货 |
| 配送中     | 已填写物流，配送中 |
| 已完成     | 管理端确认完成或签收 |
| 已取消     | 用户取消、超时关单或售后退款关闭 |

**主流程**：下单 → 待支付 → 支付成功 → 待发货 → 管理端发货 → 配送中 → 管理端确认完成 → 已完成。  
**取消分支**：待支付可被用户/超时/管理端关单 → 已取消；待发货/配送中若用户申请「仅退款」且管理端通过或完成 → 订单同步变为已取消。

### 3.2 售后类型与状态

| 售后类型        | 说明     |
|----------------|----------|
| 仅退款 REFUND_ONLY   | 不退货，只退款项 |
| 退货退款 RETURN_REFUND | 退货并退款 |
| 换货 EXCHANGE       | 换货     |

| 售后状态   | 说明     |
|-----------|----------|
| APPLIED   | 用户已申请 |
| APPROVED  | 管理端已通过 |
| REJECTED  | 管理端已拒绝 |
| COMPLETED | 售后已完成 |
| CLOSED    | 已关闭   |

### 3.3 售后与订单联动

- **仅退款**：用户在小程序订单详情对某订单项发起「仅退款」→ 管理端在「售后订单」中通过或完成该售后单时，**原订单若为待发货或配送中，会自动变为已取消**，并记录取消时间，避免订单仍显示「配送中」。
- **退货退款 / 换货**：当前仅更新售后单状态，不自动变更订单状态；后续可按业务扩展（如退货完成并退款后关单）。

### 3.4 操作入口小结

| 端     | 入口与操作 |
|--------|------------|
| 小程序 | 订单详情 → 某商品「申请售后」→ 选类型（仅退款/退货退款/换货）→ 提交；售后详情页查看进度与状态 |
| 管理端 | 售后订单 → 筛选/查看详情 → 处理售后（通过/拒绝/完成）；仅退款通过或完成时，对应订单自动变为已取消 |
