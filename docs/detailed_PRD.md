## 2. 产品背景与目标

### 2.1 背景
随着移动互联网的普及，私域流量运营成为商家增长的关键。本项目旨在为商家打造一套独立、可控的微信小程序商城系统，实现从商品展示、交易转化到售后服务的全流程闭环，帮助商家沉淀用户资产，提升经营效率。

### 2.2 目标
- **构建核心交易闭环**：实现商品浏览、加购、下单、支付、发货、收货的完整流程。
- **提供高效管理工具**：通过管理后台实现商品、订单、营销活动的便捷管理。
- **保障系统稳定性**：确保在高并发场景下（如秒杀活动）系统的稳定运行。
- **优化用户体验**：提供流畅的小程序操作体验，降低用户购买门槛。

## 3. 用户角色与权限

### 3.1 C端用户 (小程序)
- **游客**：浏览商品、查看详情、搜索商品、查看公告/Banner。
- **注册用户**：除了游客权限外，还包括：
    - 购物车管理（添加、删除、修改数量）。
    - 订单管理（下单、支付、取消、确认收货、申请售后）。
    - 个人中心管理（收货地址、查看优惠券）。

### 3.2 B端管理员 (Web后台)
- **超级管理员**：拥有系统所有权限。
- **运营人员**：
    - 商品管理（上下架、编辑、库存调整）。
    - 订单管理（发货、处理售后、查看订单详情）。
    - 营销管理（配置优惠券、Banner）。
    - 用户管理（查看用户列表、详情）。

## 4. 功能模块详解

### 4.1 小程序端 (C端)

#### 4.1.1 首页
- **Banner轮播**：展示后台配置的营销活动图，支持点击跳转至商品详情或分类页。
- **搜索入口**：顶部固定搜索栏，点击进入搜索页。
- **快捷导航 (Grid)**：展示核心品类或活动入口（如“热销榜”、“新品首发”）。
- **商品推荐**：瀑布流展示推荐商品，支持下拉加载更多。

#### 4.1.2 分类页
- **一级分类**：左侧侧边栏展示一级分类。
- **二级分类/商品**：右侧展示选中一级分类下的二级分类或直接展示商品列表。
- **交互**：点击左侧分类，右侧内容联动更新。

#### 4.1.3 商品详情页
- **基础信息**：商品轮播图、名称、价格（现价/原价）、销量、库存。
- **规格选择**：虽然目前数据模型偏向单规格（SPU=SKU），但界面预留规格选择交互。
- **图文详情**：富文本展示商品详细介绍。
- **底部栏**：
    - 客服入口（唤起微信客服）。
    - 购物车入口（带角标）。
    - 加入购物车（弹窗选择数量）。
    - 立即购买（直接跳转结算页）。

#### 4.1.4 购物车
- **列表展示**：按店铺（当前单店铺）分组展示商品。
- **编辑模式**：支持修改购买数量、删除商品。
- **选中逻辑**：支持单选、全选；计算选中商品总价。
- **状态同步**：登录后自动同步本地购物车至服务端。

#### 4.1.5 结算页 (下单)
- **地址选择**：默认展示默认地址，点击可切换或新增地址。
- **商品清单**：展示本次结算的商品及数量。
- **优惠券**：展示可用优惠券列表，自动选择最优或手动选择。
- **金额明细**：商品总额 - 优惠金额 + 运费 = 实付金额。
- **备注**：用户填写订单备注。

#### 4.1.6 订单中心
- **订单列表**：全部、待支付、待发货、配送中、已完成、已取消。
- **订单详情**：展示订单状态进度、物流轨迹（模拟）、商品信息、费用明细、支付时间等。
- **操作**：
    - 待支付：去支付、取消订单。
    - 待发货：提醒发货（仅做记录）。
    - 配送中：确认收货、查看物流。
    - 已完成/待发货：申请售后。

#### 4.1.7 售后服务
- **申请售后**：选择售后类型（仅退款、退货退款、换货）、填写原因、上传凭证。
- **售后详情**：查看审核进度、商家回复、退款状态。

#### 4.1.8 个人中心
- **用户信息**：头像、昵称。
- **我的订单**：各状态订单入口。
- **我的服务**：地址管理、优惠券、联系客服、关于我们。

### 4.2 管理后台 (B端)

#### 4.2.1 商品管理
- **商品列表**：分页展示，支持按名称、分类、状态筛选。
- **发布商品**：
    - 填写名称、副标题、价格、库存。
    - 上传主图、轮播图。
    - 编辑富文本详情。
    - 设置分类、排序权重。
- **库存管理**：快速调整商品库存。

#### 4.2.2 订单管理
- **订单查询**：支持订单号、手机号、下单时间、状态组合查询。
- **订单详情**：查看收货人信息、商品清单、支付信息。
- **发货处理**：录入快递公司与单号，变更状态为“配送中”。
- **关闭订单**：对未支付订单进行手动关闭。

#### 4.2.3 售后管理
- **售后列表**：查看所有售后申请。
- **审核处理**：
    - 同意：系统自动处理退款（对接支付退款接口）或进入退货流程。
    - 拒绝：填写拒绝原因，反馈给用户。

#### 4.2.4 营销管理
- **优惠券管理**：创建满减券，设置面额、门槛、有效期、发放总量。
- **Banner管理**：配置小程序首页轮播图。

#### 4.2.5 用户管理
- **用户列表**：查看注册用户基本信息、消费统计。

## 5. 业务流程与规则

### 5.1 订单状态机

| 状态值 (Status) | 描述 | 前置状态 | 触发动作 | 后置状态 |
| :--- | :--- | :--- | :--- | :--- |
| `PENDING_PAY` | 待支付 | - | 用户下单 | `PENDING_PAY` |
| `PENDING_SHIP` | 待发货 | `PENDING_PAY` | 用户支付成功 | `PENDING_SHIP` |
| `SHIPPED` | 配送中 | `PENDING_SHIP` | 管理员发货 | `SHIPPED` |
| `COMPLETED` | 已完成 | `SHIPPED` | 用户确认收货/系统自动收货 | `COMPLETED` |
| `CANCELLED` | 已取消 | `PENDING_PAY` | 用户取消/超时未付/管理员关闭 | `CANCELLED` |
| `CANCELLED` | 已取消 | `PENDING_SHIP`/`SHIPPED` | 售后全额退款完成 | `CANCELLED` |

### 5.2 库存扣减规则
- **方案**：下单预占（锁定库存），支付成功实扣（扣减库存）。
- **超时释放**：订单超时未支付（如30分钟），系统自动释放锁定库存。
- **取消释放**：用户主动取消或管理员关闭订单，释放锁定库存。

### 5.3 优惠券使用规则
- **领取限制**：每人限领张数配置。
- **使用限制**：单笔订单限用一张，满足门槛金额。
- **退回规则**：订单在发货前取消（全额退款），优惠券退回用户账户；发货后退款，优惠券不退回。
